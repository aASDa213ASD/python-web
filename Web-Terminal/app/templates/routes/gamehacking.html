First you will get informed about what game hacking even is:

This field is insanely large and has tons of hardships.


Game hacking can be separated into two main categories:

1. Cheats.

2. Exploits.


Cheat represents an action that a player can perform within the game

but this action is done programmatically instead.


Exploit represents the complete break of the rules,

an action that normal player can not do.

- Now you know what game-hacking is, but how it's done?

The way we hack can be separated as well: Internal/External.

Difference is huge, on external side you create a program

that runs along with target process and read it's memory.

On the other hand, internal side requires you to inject

a dynamic library inside target's memory space.


You will need a help of math if you want to build something worthy.

But only math is not enough, you have to understand how target process

is built inside, for that you have to understand assembly.

You need to know which function to call, with what arguments.

Otherwise, memory will corrupt and process will eventually die.

- In other words, this world consumed my brain entirely.

From now on you can fully understant the complexity of hacking

and I remember days when I thought "C++ is hard"

but little I knew that game hacking is even harder.
